Campus Buddy — Microservices Ecosystem

"A decoupled, multi-service campus management platform designed to automate academic workflows, facility logistics, and real-time communication through a unified Backend-for-Frontend (BFF) architecture."

The Gap

Traditional campus portals suffer from monolithic fragility and fragmented data silos. Students navigate multiple logins for attendance, notices, and facilities, while faculty face manual overhead in record-keeping. Campus Buddy bridges this gap by unifying these microflows into a single, scalable digital ecosystem.

Manual Overhead

Faculty spend 15% of lecture time on manual attendance. Campus Buddy's automated QR-tracking recovers this lost academic time.

Resource Underutilization

Campus facilities (labs, seminar halls) often sit empty due to lack of a real-time booking visibility. Our solution provides live availability tracking.

System Architecture

Auth Service

JWT-based secure authentication for students, faculty, and administrators.

Academic Service

Manages attendance logic, personalized timetables, and record persistence.

Campus Service

Real-time notice board and facility booking with H2/PostgreSQL reliability.

BFF Gateway

Backend-for-Frontend pattern orchestrating cross-service API requests.
